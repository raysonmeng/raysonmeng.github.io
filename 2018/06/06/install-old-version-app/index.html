使用 Mac 安装旧版本 iOS 应用（一）

航海日志
2018-06-06

1. 安装带有 App 管理功能的 iTunes

新版本的 Itunes 已经移除了下载 iOS App 的功能

好在苹果良心，在官网中保留了带 app 安装功能的 Itunes 的下载，需要我们手动下载安装

Download URL：https://support.apple.com/zh-cn/HT208079

安装 旧版 iTunes 后，连接手机就可以看到下图

2. 下载安装 Charles

3. 配置 Charles

3.1 打开 Charles 第一步先安装证书

  2. 点击添加
  3. 搜索证书名字关键字，找到刚刚安装的证书
  4. 设置证书为始终信任，设置后关闭此窗口提示输入密码保存设置

3.2 配置代理地址

确认【系统偏好设置 —> 网络 —> 高级 —> 代理】中勾选了【网页代理】【安全网页代理】并且不能勾选【自动代理配置】，其端口应和 Charles 【Proxy —> Proxy Settings】中设置的端口一致

3.3 开启 Charles 代理

3.4 验证是否开启

勾选了【macOS Proxy】后，点击【Proxy —>Start Recording】就可以看到左侧边栏中会有一堆网络请求出现。

4. 在 iTunes 中搜索并下载你要安装的 App

这次下载只是为了找到给加断点的请求，我以 Step Two 为例，因为这货欺负我手机系统版本不是最新的 iOS 11.4，但是这个两步验证工具颜值真的高，不安上不死心，所以就有了这个教程。

点击下载后iTunes 会自动开始下载App，同时 Charles 中会出现这个 https://xxx-buy.itunes.apple.com(xxx不绝对是 p36)

在这个请求上点右键

  1. 选择【Focus】只看这一个请求，更清晰，减少干扰
  2. 再选择【Enable SSL Proxying】对 HTTPS 解析
  3. 再选择【Breakpoints】增加断点
  4. OK，都设置完是下面这样。

5. 在 iTunes 中选择资料库，删除你刚才下载的 App

6. 重新搜索并且下载你要下载的 App 【Step Two】

由于设置了断点，Charles 中会弹出被断点拦住的请求，这是第一次弹出来，我们直接点【Execute】。

然后会再次拦截到这次请求的返回值，在这个返回值中，点击上面的【Edit Response】再选择下面的【XML Text】然后在这个返回值里面找softwareVersionExternalLdentitiers就是红框的部分，下面的数字就是这个App现在可下载的 几个版本的 id，从上到下是从老版本到新版本。把这些 id 记录下来，后面会用到。然后点击下面的【Abort】终止这次请求。

现在就拿到了这个 App 的所有历史版本id

7. 再次重新下载 App

在 Charles 中会再次弹出请求，这时候我们就修改这个请求，让他去下载老版本的 App。

  1. 点击【Edit Request】
  2. 选择【XML Text】
  3. 将【appExtVrsId】下面的值修改为你想下载的版本 id，我们这里先尝试下倒数第二个版本，也就是上一个版本的【Step Two】是不是强制要求 iOS 11.3.4 才能安装，所以就把这里修改为【827311430】
  4. 点那个红色的六边形按钮取消掉断点
  5. 【Execute】释放请求

8. 等待下载完成

下载完成后如果你看到你的 iTunes 里面的下载按钮变成了【Update】按钮，那个么恭喜你👏👏👏，成功了。

这时候就可以去【资料库】中看到下载下来的旧版本 App 了。在 App 上点击右键，选择在 Finder 中显示，就找到下载下来的【ipa】文件了。
